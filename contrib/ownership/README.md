# OpenTela Ownership Tools

Utilities for managing the OpenTela SPL ownership token on Solana. This toolkit provides everything needed to create, mint, transfer, burn, and verify SPL tokens used as proof-of-ownership for OpenTela network providers.

## Overview

The OpenTela ownership model uses an SPL token on Solana to prove that a node operator is a legitimate provider in the network. Holding a minimum balance of the ownership token grants the right to register services and participate in the decentralised network.

This toolkit provides standalone scripts and a reusable library for managing the full lifecycle of the ownership token:

| Script | Description |
|---|---|
| `create-token` | Create a new SPL token mint with on-chain Metaplex metadata |
| `mint-tokens` | Mint tokens to a wallet (requires mint authority) |
| `transfer-tokens` | Transfer tokens between wallets |
| `burn-tokens` | Permanently burn tokens from a wallet |
| `verify-ownership` | Check if wallet(s) meet the minimum token balance |
| `create-ata` | Pre-provision Associated Token Accounts for wallets |
| `revoke-mint` | Permanently disable minting (irreversible) |
| `snapshot-holders` | Take a complete snapshot of all token holders |
| `token-info` | Display detailed information about the token |

## Prerequisites

- **Node.js** ≥ 18
- **Yarn** or **npm**
- A Solana keypair (generated by `solana-keygen`, the Solana CLI, or `otela wallet create`)
- SOL for transaction fees (use devnet for testing)

## Quick Start

### 1. Install dependencies

```sh
cd tools/ownership
bun install
# or: npm install
```

### 2. Configure your environment

```sh
cp .env.example .env
```

Edit `.env` with your settings. At minimum you need:

```env
SOLANA_CLUSTER=devnet
KEYPAIR_PATH=~/.config/solana/id.json
```

If you've already created an OpenTela wallet, the toolkit will automatically discover it from `~/.config/opentela/accounts/`.

### 3. Fund your wallet (devnet)

```sh
solana airdrop 2 --url devnet
# or use the OpenTela CLI:
otela wallet airdrop 2
```

### 4. Create the ownership token

```sh
npx tsx src/create-token.ts
```

This will:
1. Create a new SPL token mint on-chain
2. Attach Metaplex metadata (name, symbol, URI)
3. Optionally mint an initial supply

After creation, copy the printed mint address into your `.env`:

```env
TOKEN_MINT=<your-mint-address>
```

### 5. Mint tokens to a provider

```sh
npx tsx src/mint-tokens.ts --to <wallet-pubkey> --amount 100
```

### 6. Verify ownership

```sh
npx tsx src/verify-ownership.ts --wallet <wallet-pubkey>
```

## Usage

### Create Token

Creates a new SPL token with the configured metadata properties.

```sh
npx tsx src/create-token.ts [options]

Options:
  --name <name>       Token name (default: from TOKEN_NAME env)
  --symbol <symbol>   Token symbol / ticker (default: from TOKEN_SYMBOL env)
  --decimals <n>      Decimal places (default: 9)
  --uri <uri>         Off-chain metadata URI
  --supply <n>        Initial supply to mint (whole tokens)
  --no-freeze         Do not set a freeze authority
  --dry-run           Print plan without sending transactions
```

**Example:**

```sh
npx tsx src/create-token.ts \
  --name "OpenTela" \
  --symbol "OTELA" \
  --decimals 9 \
  --supply 1000000
```

### Mint Tokens

Mints tokens to a specified wallet. The caller must be the current mint authority.

```sh
npx tsx src/mint-tokens.ts [options]

Options:
  --to <pubkey>       Recipient wallet (default: authority wallet)
  --amount <n>        Amount in whole tokens (required)
  --mint <pubkey>     Token mint override
  --raw               Treat amount as raw integer
  --dry-run           Print plan without sending transactions
```

**Examples:**

```sh
# Mint 1000 tokens to a provider
npx tsx src/mint-tokens.ts --to 5YNmS1R9nNSCDzb5a7mMJ1dwK9uHeAAF4CerBTpfXoA4 --amount 1000

# Mint to yourself
npx tsx src/mint-tokens.ts --amount 500

# Mint a raw amount (useful for precise sub-decimal amounts)
npx tsx src/mint-tokens.ts --to <pubkey> --amount 1500000000 --raw
```

### Transfer Tokens

Transfers tokens from the authority wallet to a recipient. The destination ATA is created automatically if needed.

```sh
npx tsx src/transfer-tokens.ts [options]

Options:
  --to <pubkey>       Recipient wallet (required)
  --amount <n>        Amount in whole tokens (required)
  --mint <pubkey>     Token mint override
  --raw               Treat amount as raw integer
  --dry-run           Print plan without sending transactions
```

**Example:**

```sh
npx tsx src/transfer-tokens.ts \
  --to 5YNmS1R9nNSCDzb5a7mMJ1dwK9uHeAAF4CerBTpfXoA4 \
  --amount 50
```

### Verify Ownership

Checks whether one or more wallets hold the required minimum token balance.

```sh
npx tsx src/verify-ownership.ts [options]

Options:
  --wallet <pubkey>   Wallet(s) to verify (repeatable)
  --file <path>       Newline-delimited file of wallet public keys
  --stdin             Read wallets from stdin
  --mint <pubkey>     Token mint override
  --min-balance <n>   Minimum required balance (default: from env)
  --json              Output as JSON
  --quiet             Only output pass/fail (exit code reflects result)
  --verbose           Show additional details
```

**Examples:**

```sh
# Check a single wallet
npx tsx src/verify-ownership.ts --wallet 5YNmS1R9nNSCDzb5a7mMJ1dwK9uHeAAF4CerBTpfXoA4

# Check multiple wallets
npx tsx src/verify-ownership.ts \
  --wallet <pubkey1> \
  --wallet <pubkey2> \
  --wallet <pubkey3>

# Check from a file
npx tsx src/verify-ownership.ts --file providers.txt

# Use in CI/CD (exit code 0 = pass, 1 = fail)
npx tsx src/verify-ownership.ts --wallet <pubkey> --quiet

# JSON output for programmatic consumption
npx tsx src/verify-ownership.ts --wallet <pubkey> --json

# Pipe wallet addresses
echo "5YNmS1R9nNSCDzb5a7mMJ1dwK9uHeAAF4CerBTpfXoA4" | npx tsx src/verify-ownership.ts --stdin
```

### Token Info

Displays comprehensive information about the token.

```sh
npx tsx src/token-info.ts [options]

Options:
  --mint <pubkey>     Token mint override
  --holders           Show top 20 holders
  --all-holders       Snapshot all holders (can be slow)
  --json              Output as JSON
```

**Examples:**

```sh
# Basic token info
npx tsx src/token-info.ts

# Include top holders
npx tsx src/token-info.ts --holders

# Full snapshot as JSON
npx tsx src/token-info.ts --all-holders --json > token-info.json
```

### Create ATA

Pre-provisions Associated Token Accounts for wallets.

```sh
npx tsx src/create-ata.ts [options]

Options:
  --wallet <pubkey>   Wallet(s) to create ATAs for (repeatable)
  --file <path>       Newline-delimited file of wallet public keys
  --stdin             Read wallets from stdin
  --mint <pubkey>     Token mint override
  --check-only        Only check existence (exit code 0 = all exist)
  --json              Output as JSON
  --dry-run           Print plan without sending transactions
```

**Examples:**

```sh
# Create ATAs for specific wallets
npx tsx src/create-ata.ts --wallet <pubkey1> --wallet <pubkey2>

# Check if ATAs exist (useful in CI)
npx tsx src/create-ata.ts --file providers.txt --check-only

# Batch create from file
npx tsx src/create-ata.ts --file new-providers.txt
```

### Burn Tokens

Permanently destroys tokens from your wallet, reducing the circulating supply.

```sh
npx tsx src/burn-tokens.ts [options]

Options:
  --amount <n>        Amount to burn (required)
  --mint <pubkey>     Token mint override
  --raw               Treat amount as raw integer
  --yes               Skip confirmation prompt
  --dry-run           Print plan without sending transactions
```

**Example:**

```sh
# Burn 100 tokens (interactive confirmation)
npx tsx src/burn-tokens.ts --amount 100

# Burn with auto-confirm (for scripts)
npx tsx src/burn-tokens.ts --amount 100 --yes
```

### Revoke Mint Authority

Permanently disables minting, locking the total supply forever. **This is irreversible.**

```sh
npx tsx src/revoke-mint.ts [options]

Options:
  --mint <pubkey>     Token mint override
  --revoke-freeze     Also revoke freeze authority
  --yes               Skip confirmation prompt
  --dry-run           Print plan without sending transactions
```

**Example:**

```sh
# Preview what would happen
npx tsx src/revoke-mint.ts --dry-run

# Revoke minting (interactive confirmation)
npx tsx src/revoke-mint.ts

# Revoke both mint and freeze authorities
npx tsx src/revoke-mint.ts --revoke-freeze --yes
```

### Snapshot Holders

Takes a complete snapshot of all token holders via `getProgramAccounts`.

```sh
npx tsx src/snapshot-holders.ts [options]

Options:
  --mint <pubkey>       Token mint override
  --min-balance <n>     Minimum balance filter (whole tokens)
  --top <n>             Only include top N holders
  --out <path>          Write to file instead of stdout
  --format <fmt>        Output format: json, csv, or table
  --json                Alias for --format json
  --csv                 Alias for --format csv
  --include-zero        Include zero-balance accounts
  --quiet               Suppress informational output
```

**Examples:**

```sh
# Full snapshot as JSON
npx tsx src/snapshot-holders.ts --out snapshot.json

# CSV for spreadsheet import
npx tsx src/snapshot-holders.ts --out holders.csv --csv

# Top 50 holders as a table
npx tsx src/snapshot-holders.ts --top 50 --format table

# Only holders with at least 100 tokens
npx tsx src/snapshot-holders.ts --min-balance 100 --json

# Pipe-friendly (quiet mode)
npx tsx src/snapshot-holders.ts --quiet --json > holders.json
```

## Configuration

All scripts read configuration from environment variables and/or a `.env` file in the `tools/ownership/` directory. See [`.env.example`](.env.example) for the full list.

### Key Variables

| Variable | Description | Default |
|---|---|---|
| `SOLANA_CLUSTER` | Cluster: `mainnet-beta`, `devnet`, `testnet`, `localnet` | `devnet` |
| `SOLANA_RPC_URL` | Custom RPC endpoint (overrides cluster) | — |
| `KEYPAIR_PATH` | Path to Solana CLI keypair JSON file | Auto-discovered |
| `PRIVATE_KEY_BASE58` | Base58-encoded private key (Phantom format) | — |
| `TOKEN_MINT` | SPL token mint address | — |
| `TOKEN_NAME` | Token name for metadata | `OpenTela` |
| `TOKEN_SYMBOL` | Token symbol / ticker | `OTELA` |
| `TOKEN_DECIMALS` | Decimal places | `9` |
| `TOKEN_URI` | Off-chain metadata URI | — |
| `TOKEN_INITIAL_SUPPLY` | Initial supply to mint on creation | `0` |
| `OWNERSHIP_MIN_BALANCE` | Minimum balance for ownership verification | `1` |
| `DEBUG` | Enable verbose / debug output | `false` |

### Keypair Discovery

The toolkit searches for a keypair in this order:

1. `KEYPAIR_PATH` environment variable
2. `PRIVATE_KEY_BASE58` environment variable
3. First managed wallet in `~/.config/opentela/accounts/`
4. Default Solana CLI keypair at `~/.config/solana/id.json`

This means if you've created a wallet with `otela wallet create`, it will be found automatically.

## Library Usage

The ownership tools can also be imported as a library for programmatic use:

```typescript
import {
  loadConfig,
  getConnection,
  createSPLToken,
  mintTokensTo,
  verifyOwnership,
  toRawAmount,
  fromRawAmount,
} from "./src/index.js";

// Load configuration from .env
const cfg = loadConfig();
const connection = getConnection(cfg);

// Create a new token
const { mint, signature } = await createSPLToken(
  connection,
  cfg.keypair,
  9, // decimals
);
console.log("Mint created:", mint.toBase58());

// Mint tokens
const rawAmount = toRawAmount(1000, 9);
await mintTokensTo(connection, cfg.keypair, mint, recipientPubkey, rawAmount);

// Verify ownership
const result = await verifyOwnership(connection, walletPubkey, mint, 1);
console.log("Is owner:", result.isOwner);
console.log("Balance:", result.uiBalance);
```

### Key Functions

| Function | Description |
|---|---|
| `createSPLToken()` | Create a new SPL token mint |
| `mintTokensTo()` | Mint tokens to a wallet |
| `transferTokens()` | Transfer tokens between wallets |
| `burnTokens()` | Burn tokens from a wallet |
| `verifyOwnership()` | Check single wallet ownership |
| `verifyOwnershipBatch()` | Check multiple wallets at once |
| `revokeMintAuthority()` | Disable minting permanently |
| `revokeFreezeAuthority()` | Disable freezing permanently |
| `transferMintAuthority()` | Hand off mint authority to another key |
| `getOrCreateATA()` | Get or create an Associated Token Account |
| `getTokenBalance()` | Query a wallet's token balance |
| `getTokenSupply()` | Query total token supply |
| `snapshotAllHolders()` | Get all token holders via `getProgramAccounts` |
| `getLargestHolders()` | Get top 20 holders via `getTokenLargestAccounts` |
| `toRawAmount()` | Convert UI amount → raw integer |
| `fromRawAmount()` | Convert raw integer → UI amount |
| `requestAirdrop()` | Request SOL from devnet/testnet faucet |

## Integration with OpenTela

### How Ownership Works

1. **Token creation**: The network operator creates an SPL token using `create-token`.
2. **Distribution**: Tokens are minted to provider wallets using `mint-tokens`.
3. **Verification**: When a node registers a service, the network verifies that the node's wallet holds the required minimum token balance.
4. **Ongoing checks**: The `verify-ownership` script (or the library function) can be used in health checks, CI/CD pipelines, or monitoring dashboards.

### Provider ID Mapping

Each OpenTela wallet has a deterministic Provider ID derived from its public key (format: `otela-<first 12 chars of base58 pubkey>`). This Provider ID is recorded in the distributed node table and links a node's identity to its on-chain wallet.

### Interoperability with `otela wallet`

The ownership tools are fully compatible with wallets created by the `otela` CLI:

```sh
# Create a wallet with otela
otela wallet create

# The ownership tools will auto-discover it
npx tsx src/verify-ownership.ts --wallet $(otela wallet info | grep "Public key" | awk '{print $3}')

# Export for use with Solana CLI
otela wallet export --file /tmp/keypair.json
KEYPAIR_PATH=/tmp/keypair.json npx tsx src/create-token.ts
```

### Anchor Program Integration

The `tokens/` directory contains an Anchor-based token minting program. While the ownership tools use the native SPL Token Program directly (no Anchor dependency required at runtime), they are compatible with tokens created by the Anchor program. Set `ANCHOR_PROGRAM_ID` in `.env` if you need the tools to reference the deployed program.

## Security Considerations

- **Never commit `.env` files** containing private keys or keypair paths to source control.
- **Keypair files** should have restrictive permissions (`chmod 600`).
- **Revoking mint authority** is permanent. Ensure the total supply is correct before running `revoke-mint`.
- **Burning tokens** is permanent. Double-check amounts before confirming.
- Use **`--dry-run`** on any script to preview what will happen without sending transactions.
- For production deployments, consider using a **hardware wallet** or **multisig** as the mint authority.
