## Overview

Implements Solana blockchain settlement for the usage tracking and billing system. Head nodes initiate SPL token (OTELA) payments from consumers to providers based on agreed-upon usage metrics and provider-set rates.

## Goals

- **Automated payments** - Head nodes orchestrate payments after service completion
- **SPL Token transfers** - Use OpenTela's custom OTELA token
- **Dynamic pricing** - Each provider sets their own rates per service/metric
- **Batch settlement** - Payments aggregated and processed periodically
- **Trustworthy** - Dual attestation, transparent rates, on-chain verification

## Architecture

Head nodes act as payment processors, building and signing SPL token transfer transactions on behalf of consumers after services are completed and usage is reconciled between head and worker nodes.

## Components

### 1. Rate Manager (`internal/solana/rates.go`)

Manages provider pricing per service and metric.

```go
type Rate struct {
    ProviderAddress string // Solana address
    Service         string // "llm", "sandbox", etc.
    MetricName      string // "tokens", "gpu_ms"
    PricePerUnit    int64  // OTELA base units (9 decimals)
}

type RateManager struct {
    rates map[string]Rate // key: "provider_address:service:metric"
}
```

**Functions:**
- `GetRate(provider, service, metric) (Rate, error)`
- `SetRate(rate) error`
- `LoadFromConfig(path) error` - Load provider rates from YAML

**Config Format (`/etc/opentela/rates.yaml`):**
```yaml
providers:
  - address: "ProviderSolanaAddress1"
    services:
      - name: "llm"
        metrics:
          - name: "tokens"
            price_per_1000: 1000  # base units
```

### 2. Settlement Builder (`internal/solana/settlement.go`)

Builds SPL Token transfer transactions.

**Key Functions:**
- `BuildSPLTransfer(from, to, amount, mint) ([]byte, error)` - Build TransferChecked instruction
- `CreateATAInstruction(owner, mint) ([]byte, error)` - ATA creation for first-time users
- `BuildSettlementTx(consumer, provider, resolvedUsage, rates) (*Transaction, error)`

**SPL Token Program Details:**
- **Program ID:** `TokenkeQ4aoQhLvy6RtXFn89jYq8RKzVKdk1` (SPL Token 2022)
- **Instruction:** `TransferChecked`
- **ATA Derivation:** `PDA(owner, mint, TOKEN_PROGRAM_ID)`

### 3. Payment Processor (`internal/solana/processor.go`)

Orchestrates the payment workflow.

```go
type PaymentProcessor struct {
    client      *Client
    rateManager *RateManager
    walletMgr   *wallet.WalletManager

type PaymentResult struct {
    Signature  string
    Amount     int64
```

**Functions:**
- `ProcessUsageRecords(ctx, records) ([]PaymentResult, error)`
- `VerifyBalance(ctx, address, requiredAmount) (bool, error)`
- `SubmitPayment(ctx, tx) (string, error)`
- `ConfirmTransaction(ctx, signature) error`

### 4. Consumer Billing Registry

Stores consumer wallet addresses for automated billing.

**CRDT Namespace:** `/billing/wallets/{peer_id}`

```go
type ConsumerWallet struct {
    PeerID        string // Head node peer ID
    SolanaAddress string // Consumer's Solana wallet
    PrivateKeyEnc string // Encrypted private key (for signing)
}
```

## Data Flow

1. **Aggregation Trigger** (value_threshold OR max_interval)
   - `Aggregator.BuildAggregate()` creates `AggregatedUsage`

2. **CRDT Sharing**
   - Head publishes aggregate via `PublishAggregate(ctx, agg)`
   - Key: `/usage/{head_id}/{service}/{metric}/{window_start}`

3. **Reconciliation** (Worker node does this)
   - `GetPeerAggregate(ctx, head_id, ...)`
   - `ReconcileRecords(headUsage, workerUsage, thresholdPct)`
   - Returns `ResolvedUsage` with agreed value or disputed flag

4. **Rate Lookup**
   - `rateManager.GetRate(provider, service, metric)`
   - Returns `PricePerUnit` in OTELA base units

5. **Calculate Amount**
   - `amount = resolvedValue Г— rate.PricePerUnit`

6. **Build Transaction**
   - Build `TransferChecked` instruction for SPL Token
   - Include both consumer and provider ATAs
   - Add recent blockhash

7. **Sign Transaction**
   - Sign with consumer's private key
   - Consumer has authorized this head node for billing

8. **Submit to Solana**
   - `client.sendTransaction(tx, {skipPreflight: false})`

9. **Confirmation**
   - Poll `getSignatureStatuses` until confirmed
   - Timeout: 60 seconds

10. **Record Payment**
    - Store in CRDT: `/billing/payments/{signature}`
    - Prevents duplicate payments

## Error Handling

| Scenario | Handling |
|----------|----------|
| Insufficient balance | Retry with backoff, notify after 3 attempts |
| Invalid ATA | Create ATA first, retry transfer |
| Network timeout | Retry up to 3 times, 30s timeout each |
| Transaction fee too low | Priority fee bump, retry |
| Consumer wallet not found | Skip payment, flag for review |
| Disputed usage (>10% diff) | Skip payment, flag for review |
| Provider rate not found | Use default platform rate, log warning |

### Idempotency

- Payment records stored by transaction signature
- Check for existing payment before submitting
- Unique request IDs prevent double-billing

## Implementation Notes

- Uses existing `Client` from `internal/solana/client.go`
- Uses existing `WalletManager` from `internal/wallet/wallet.go`
- Extends existing `settlement.go` skeleton
- Consumer private keys must be accessible to head node (encrypted storage)
- First payment to a provider requires ATA creation (budget for ~0.002 SOL)
